# Интеграция датчиков дверей в SNMP-агент

## Выполненные изменения

### 1. Исправление логики датчиков
В файле `sensors_app.c` исправлена инверсия логики для функций `get_door_open_status_1` и `get_door_open_status_2`.
- Теперь функции возвращают `1` (Alarm/Open), когда физический контакт разомкнут (пин в состоянии `1` благодаря подтяжке PULLUP).
- Это соответствует схеме подключения из `hardware_analysis.md`, где закрытая дверь замыкает геркон на землю.

### 2. Интеграция в SNMP MIB
В файле `main.c` подтверждено наличие и корректность обработки OID для дверей:
- `.1.3.6.1.4.1.9999.1.7.0` — Статус двери 1.
- `.1.3.6.1.4.1.9999.1.8.0` — Статус двери 2.
Значения теперь корректно возвращаются через SNMP callback `get_system_metrics`.

### 3. Обновление документации
Файл `project_overview.md` обновлен до актуального состояния. Теперь он содержит:
- Новые константы GPIO для дверей.
- Обновленные функции в `sensors_app.c`.
- Актуальный код `main.c` с поддержкой новых OID.
- Прототипы функций в `sensors_app.h`.

## Верификация
- **Анализ кода:** Проведена проверка соответствия вызовов функций и определений констант.
- **Логическая проверка:** Подтверждено соответствие логики работы ПО и аппаратной схемы (`0` = закрыто, `1` = открыто).

> [!NOTE]
> Сборка проекта через `idf.py` не была выполнена из-за отсутствия настроенного окружения в терминале, однако код прошел ручную проверку на отсутствие синтаксических и логических ошибок.
