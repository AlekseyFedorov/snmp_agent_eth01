В иерархии SNMP группа system всегда находится по адресу .1.3.6.1.2.1.1.
```
Основные узлы этой группы:
- sysDescr.1.3.6.1.2.1.1.1.0Описание устройства (по умолчанию LwIP)
- sysUpTime.1.3.6.1.2.1.1.3.0Системный аптайм (стандартный)
- sysContact.1.3.6.1.2.1.1.4.0То, что вы записали в syscontact_storage
- sysName.1.3.6.1.2.1.1.5.0Имя узла (обычно "esp32" или имя хоста)
- sysLocation.1.3.6.1.2.1.1.6.0То, что вы записали в syslocation_storage
```


## Изменение сетевых параметров

1. Изменение шлюза (Gateway)
Рекомендуется делать это первым, чтобы устройство не ушло в перезагрузку до того, как вы успеете отправить всё остальное.

```bash
snmpset -v 2c -c private 192.168.2.50 .1.3.6.1.4.1.9999.1.11.0 s "192.168.2.1"
```
2. Изменение маски подсети (Netmask)
Эту команду также можно отправить в любой момент.

```bash
snmpset -v 2c -c private 192.168.2.50 .1.3.6.1.4.1.9999.1.12.0 s "255.255.255.0"
```
3. Изменение самого IP-адреса
Лучше всего отправлять эту команду последней.

```bash
snmpset -v 2c -c private 192.168.2.50 .1.3.6.1.4.1.9999.1.10.0 s "192.168.2.55"
```
## Пакетная отправка (Всё сразу — рекомендуемый способ)
Поскольку любая команда snmpset в вашем коде вызывает функцию ребута, изменение параметров по одному может быть неудобным (вам придётся отправлять, ждать перезагрузки платы и отправлять следующую).

Протокол SNMP позволяет отправить все новые значения в одном пакете (одним запросом):

```bash
snmpset -v 2c -c private 192.168.2.50 \
  .1.3.6.1.4.1.9999.1.10.0 s "192.168.2.55" \
  .1.3.6.1.4.1.9999.1.11.0 s "192.168.2.1" \
  .1.3.6.1.4.1.9999.1.12.0 s "255.255.255.0"
```
