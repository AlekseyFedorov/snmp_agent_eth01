# SNMP Agent на базе WT32-ETH01

Мониторинг серверного помещения по SNMP v2c: температура, протечка воды, открытие дверей, удалённая смена сетевых настроек.

## Возможности

- **Температура** — DS18B20 по 1-Wire, опрос каждые 2 секунды
- **Протечка воды** — Gidrolock WSP через транзисторную развязку
- **Датчики дверей** — 2 геркона (сухой контакт)
- **Светодиодная индикация** — сеть, авария
- **SNMP v2c** — стандартный MIB2 + кастомный enterprise MIB
- **Удалённая смена IP/шлюза/маски** — SNMP SET с валидацией и авто-перезагрузкой
- **Сброс к заводским** — удержание кнопки 5 секунд (стирает NVS)
- **Аппаратный watchdog** — автоматическая перезагрузка при зависании задач

## Логика работы датчиков

### DS18B20 (температура)

Питание 3.3V, данные по 1-Wire на GPIO 32. Между VDD и DQ обязателен резистор 4.7 кОм. Значение публикуется в SNMP как `INTEGER` (температура × 10).

### Gidrolock WSP (протечка)

Датчик питается от 5V. Без воды — ток не течёт, база транзистора (2N3904) притянута к GND резистором 10 кОм, транзистор закрыт, на GPIO 33 — логическая 1 (PULLUP). При протечке вода замыкает контакты, ток через резистор базы 10 кОм открывает транзистор, GPIO замыкается на GND — логический 0.

> [!CAUTION]
> Никогда не подключайте Gidrolock WSP напрямую к GPIO! Напряжение шлейфа 5–24V сожжёт порт. Только через транзистор.

### Герконы (двери)

Дверь закрыта: магнит рядом, геркон замкнут, GPIO → GND = 0. Дверь открыта: геркон разомкнут, GPIO → PULLUP = 1. SNMP возвращает 1 при открытии (тревога).

## Сборка и прошивка

### Требования

- ESP-IDF v5.0+ (протестировано на v6.0.1)
- LWIP с поддержкой SNMP (включается автоматически через компонент `snmp_lwip`)

### Порядок сборки

```bash
git clone https://github.com/AlekseyFedorov/snmp2_agent_eth01.git
cd snmp2_agent_eth01

idf.py set-target esp32
idf.py build
idf.py -p COM3 flash monitor
```

### Заводские параметры

| Параметр | Значение |
|----------|---------|
| IP-адрес | 10.149.130.75 |
| Шлюз | 10.149.130.65 |
| Маска | 255.255.255.224 |
| Flash | 8 MB |
| SNMP Read Community | public |
| SNMP Write Community | private |

> Параметры хранятся в `main/ethernet_app.c` и NVS. После первого запуска сохраняются в NVS. При изменении через SNMP SET устройство перезагружается.

## SNMP-команды

Базовый OID: `.1.3.6.1.4.1.9999.1`

### Чтение (GET)

```bash
# Температура (×10°C)
snmpget -v 2c -c public <IP> .1.3.6.1.4.1.9999.1.1.0

# Протечка (0 — сухо, 1 — тревога)
snmpget -v 2c -c public <IP> .1.3.6.1.4.1.9999.1.6.0

# Дверь 1 (0 — закрыта, 1 — открыта)
snmpget -v 2c -c public <IP> .1.3.6.1.4.1.9999.1.7.0

# Дверь 2
snmpget -v 2c -c public <IP> .1.3.6.1.4.1.9999.1.8.0

# Пройти всё дерево
snmpwalk -v 2c -c public <IP> .1.3.6.1.4.1.9999.1
```

### Запись (SET)

```bash
# Изменение IP/шлюза/маски (одним пакетом):
snmpset -v 2c -c private <IP> \
  .1.3.6.1.4.1.9999.1.10.0 s "10.149.130.80" \
  .1.3.6.1.4.1.9999.1.11.0 s "10.149.130.65" \
  .1.3.6.1.4.1.9999.1.12.0 s "255.255.255.224"
```

После записи устройство перезагрузится через 1 секунду.

## Индикация

| Светодиод | GPIO | Логика |
|-----------|------|--------|
| LED Сеть (зелёный) | IO2 | Горит — сеть активна; мигает — нет линка |
| LED Авария (красный) | IO4 | Мигает при: протечке, открытой двери, t > 45°C |

## Сброс к заводским настройкам

Удержание кнопки на GPIO 5 в течение 5 секунд стирает NVS-раздел. После перезагрузки устройство стартует с параметрами по умолчанию из `ethernet_app.c`.

## Возможные улучшения

- **DHT22 / BME280** — датчик влажности и давления для полного климат-контроля
- **Оптрон PC817** — гальваническая развязка датчика протечки вместо транзистора
- **SNMP Trap** — активная отправка уведомлений в NMS (Zabbix/PRTG) при авариях
- **mDNS** — авто-обнаружение устройства в сети (`snmp-agent.local`)
